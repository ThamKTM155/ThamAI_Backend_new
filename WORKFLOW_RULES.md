# Workflow Rules

## Nguyên tắc phát triển hệ thống

1. Ưu tiên production ổn định
2. Không refactor lớn khi hệ đang chạy tốt
3. Test sau mỗi thay đổi
4. Commit thường xuyên
5. Backup trước khi nâng cấp lớn
6. Ghi nhật ký mỗi phiên làm việc
7. Tài liệu hóa deployment và architecture

---

# Quy trình chuẩn

Code
→ Test Local
→ Commit GitHub
→ Deploy
→ Test Production
→ Ghi chú checkpoint
